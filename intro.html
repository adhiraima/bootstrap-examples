Best Practices
  • Please DO NOT USE in-line styling. Use Precompilers and or externalize all the style in a single file which can be duplicated for theme-ing without extra effort.
  • Please USE IDs for every element possible on the page. If you use angular, please use IDs in the component templates or the template in-line in the TS file.
  • Please write unit tests for the components and services in the front-end code. Please estimate with the buffer for writing tests.
  • Please use proper naming conventions. e.g. a formID is formXYZ then the name i/p field should have the id formXYZName. Or you are free to choose any convention that is logical and clear.
    Remember the http request uses the name to make the param name not ID. ID is only for the DOM. This make the code easy to test using tools like selenium etc.
  • Please use a lint tool in the IDE, this will help in keeping the code clean SAMPLE
